Yes—Traefik absolutely supports pattern-based matching, and your use case is actually a very natural fit for its routing model. What you’re describing is essentially a two-tier Traefik architecture with dynamic routing based on host patterns, and it can be done cleanly without downtime.

✅ 1. Pattern-based matching in Traefik

Traefik supports flexible rule syntax like:

Host()

HostRegexp()

PathPrefix()

For your case, the key feature is:

🔹 HostRegexp

Example:

rule: "HostRegexp(`{workspace:[a-z0-9-]+}-{service:[a-z0-9-]+}-{stage:[a-z0-9-]+}.example.com`)"

This lets Traefik:

Match dynamic subdomains

Extract variables (workspace, service, stage)

Route based on patterns without predefining every hostname

🏗️ 2. Recommended architecture

You’re already thinking in the right direction. Here’s a clean design:

🌍 Global Traefik (edge)

Responsibilities:

TLS termination (*.hostname)

Public entrypoint (:443)

Route to correct workspace-stage Traefik

Routing strategy

Use HostRegexp to forward to the correct internal Traefik:

http:
  routers:
    workspace-router:
      rule: "HostRegexp(`{workspace}-{service}-{stage}.example.com`)"
      service: workspace-proxy
      entryPoints:
        - websecure
      tls: {}

  services:
    workspace-proxy:
      loadBalancer:
        servers:
          - url: "http://internal-traefik-router"

But here’s the catch:

👉 You cannot dynamically compute backend URLs purely from regex captures in Traefik.

So instead, you need service discovery.

🔑 3. The key: dynamic service discovery

To make this work without downtime, you should rely on a provider like:

Docker provider (most common)

Kubernetes (if applicable)

File provider (less dynamic)

🐳 With Docker (recommended)

Each workspace-stage Traefik:

Registers itself with labels

Gets picked up automatically by global Traefik

Example labels on each workspace Traefik:

labels:
  - "traefik.enable=true"
  - "traefik.http.routers.ws1.rule=HostRegexp(`ws1-.*-dev.example.com`)"
  - "traefik.http.services.ws1.loadbalancer.server.port=80"

But better:

👉 Let global Traefik route per workspace-stage, not per service.

🧠 4. Clean routing model (important)

Instead of routing per service at the global level:

✅ Route per workspace-stage Traefik

Global Traefik:

{workspace}-{service}-{stage}.example.com
        ↓
{workspace}-{stage}-traefik (internal)
        ↓
service
🧩 5. How to map requests correctly
Option A (simple & scalable): wildcard routing + shared network

Each workspace-stage Traefik:

Connected to a shared network with global Traefik

Has a predictable internal DNS name like:

traefik-ws1-dev
traefik-ws2-prod

Then global Traefik uses multiple routers generated dynamically via Docker labels:

labels:
  - "traefik.http.routers.ws1-dev.rule=HostRegexp(`ws1-.*-dev.example.com`)"
  - "traefik.http.services.ws1-dev.loadbalancer.server.port=80"
🚀 6. Zero-downtime workspace creation

This is where Traefik shines:

New workspace spins up → registers via Docker labels

Global Traefik auto-detects → routing updated instantly

No reloads, no downtime

⚠️ 7. Networking constraint (important)

You said:

workspaces and stages must have separate networks

That’s fine—but:

👉 Each workspace Traefik must also connect to a shared “edge” network so global Traefik can reach it.

Example:

edge-network (shared)

ws1-dev-network (isolated)

Workspace Traefik connects to both.

Services only connect to:

ws1-dev-network

🔐 8. TLS handling (you’re correct)

Global Traefik:

Handles TLS (*.hostname)

Internal Traefik:

Plain HTTP only

This is standard and efficient.

🧱 9. Alternative (more advanced)

If you want fully dynamic routing without per-workspace config:

Use a custom service registry

Or Traefik + plugin / forward auth

Or switch to something like:

Kubernetes with Ingress

But for Docker setups, labels are usually enough.

✅ Final verdict

Yes, your architecture works and is commonly used.

The key principles:

Use HostRegexp for pattern matching

Route to workspace Traefik, not services

Use Docker provider for dynamic config

Use a shared edge network

Let per-workspace Traefik handle final routing
